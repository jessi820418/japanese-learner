# 例句固定格式與 per-level 用字規範

本文件定義 `data/*.json` 中每筆 `examples[]` 的產出格式與各等級（N5/N4/N3）的用字界線。
`scripts/validate-examples.cjs` 會依本文件的硬性規則做機器檢查；無法機器檢查的（用字超綱、語意正確）需人工校對。

## 1. 例句 schema

單字與文法共用同一結構（`src/types/index.ts` 的 `Example`）：

```jsonc
{
  "sentence": "{私|わたし}【は】{学生|がくせい}【です】。", // 日文，含標注
  "chinese":  "我是學生。"                                  // 繁體中譯
}
```

### 標注語法

- `{kanji|reading}` — 手動 furigana。漢字部分顯示本體，`reading` 僅供標音。
  例：`{私|わたし}` 顯示「私」，上方標「わたし」。
- `【...】` — 文法重點標記。在文法卡會以 `GrammarHighlight` 標色，在「填空」模式會被挖空。
  單字例句若要標出目標單字，也可用 `【...】` 包住該詞。
- 兩種標注可交錯、可巢狀，順序不限。

## 2. 固定產出格式

validator 的嚴格度依**類別**而定：單字例句由我們依規格生成，採硬性規則；文法例句是既有教材、句子自然較長且含教學註解，故長度/句數/句尾為警告。

**永遠是硬性錯誤（單字＋文法皆擋，`--strict` 會失敗）：**

| 規則 | 說明 |
| --- | --- |
| 中譯 | 一律有 `chinese`，且為繁體中文、非空白 |
| 標記合法 | `{}`、`【】` 必須成對且可被 parser 解析（`{kanji\|reading}` 需含 `\|`） |
| 文法必有例句 | 文法項目 `examples` 不可為空 |

**單字為錯誤、文法為警告（依類別）：**

| 規則 | 單字 | 文法 |
| --- | --- | --- |
| 句長（去標注與教學註解後的「核心句」字元數） | ≤ 20 字 | ≤ 40 字（超過僅警告） |
| 句尾標點 | 須以「。」「？」「！」結尾 | 同左（僅警告） |
| 句數 | ≤ 2 句 | ≤ 8 句（僅警告） |

> 「核心句」= 純日文（去 `{}` furigana 與 `【】`）再移除教學註解：全形括號 `（原句：…）`、`→…說明`。
> 含目標：例句應包含該筆的目標單字／文法（建議用 `【...】` 標出；此項不機器強制）。

## 3. Per-level 用字規範（人工校對界線）

原則：**例句只能用「不超過該等級」的詞彙與文法**，讓學習者在當前程度即可讀懂。

- **N5**：只用 N5 範圍的詞彙與文法。句型以「は／が／を／に／で」基本助詞、です・ます、基本動詞變化為主。避免使用 N4 以上文法（如使役、受身、條件 たら・ば 的進階用法）。
- **N4**：可用到 N4（含 N5）。允許 て形連接、普通形、意向形、可能形、condition（たら／ば／と／なら）、授受表現等。
- **N3**：可用到 N3（含 N4、N5）。允許敬語、使役受身、より進階的接續詞與書面語。

> 跨等級引用：N5 例句不得出現只在 N4/N3 才教的詞或文法；反之高等級可向下相容。
> validator 無法自動判斷「超綱」，此項需人工或外部詞表比對；validator 只會在 `--strict` 下對明顯超長/缺中譯/標記錯誤報錯。

## 4. 重建流程（建議）

1. `node scripts/migrate-examples.cjs` — 把單字 `full_explanation` 內的「例：」句抽成結構化 `examples`（中譯留空）。
2. 用 LLM 批次生成 / 補齊 `chinese` 與（必要時）重寫例句，使其符合本格式與 per-level 規範。
   - 建議獨立 script：讀舊資料 → 輸出新結構，方便重跑。
   - LLM prompt 請附上本文件第 2、3 節作為約束。
3. `node scripts/validate-examples.cjs --strict` — 檢查格式；修到全綠。
4. 同步更新 `e2e/fixtures/*.json`。
